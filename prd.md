## 1 产品目标与成功指标

### 1.1 产品愿景

利用 Three.js（WebGPU / WebGL）在浏览器端复刻经典街机《雷电 II》的核心玩法。不是简单的 2D 模仿，而是通过 3D 渲染引擎带来的深度感、动态光影与物理特效，打造一款 **2.5D 现代化弹幕射击游戏**。

目标平台：

- Desktop 浏览器（Chrome / Edge / Safari）
- Mobile 浏览器
- 可封装为 PWA 或 Electron

### 1.2 核心产品目标

打造一个经典街机玩法 × 3D 视觉升级 × 即开即玩 × 高帧率的现代化网页弹幕射击游戏。

### 1.3 成功指标

| 指标 | 目标值 |
| --- | --- |
| 帧率 | 稳定 60 FPS |
| 输入延迟 | < 16 ms |
| 启动加载时间 | < 3 秒 |
| 包体大小 | < 10 MB |
| 一局时长 | 5 ～ 15 分钟 |

## 2 世界观与背景故事

### 2.1 背景故事

2090 年，地球遭到名为**克拉纳斯（Cranassians）**的异星生命体入侵。这种生命体具备半生物结构、自适应机械化能力和寄生地球军事设施的特性，能够快速接管人类的坦克、航母、战机与防空炮。

人类残余部队开发出了最终反击兵器——**雷电战机 Fighting Thunder ME-26**，特点包括 AI 辅助瞄准系统、高能激光核心和轨道核弹武器。玩家是最后的飞行员。

### 2.2 总体艺术方向

**风格关键词：** 写实军事 · 工业科幻 · 战争废墟 · 重金属机械

**视觉参考：** 雷电 II、Ace Combat、Halo 战场风格

### 2.3 3D 视觉表现要点

| 表现元素 | 实现方式 |
| --- | --- |
| 地面阴影 | 战机实时阴影投射，阴影随太阳方向变化 |
| 云层遮挡 | 战机穿越云层，云雾粒子特效 |
| 景深 | 远景模糊、近景清晰（Depth of Field） |
| 动态光照 | 激光与爆炸产生瞬时 PointLight |
| 视差滚动 | cloudSpeed=0.4 / groundSpeed=1.0 / seaSpeed=0.8 多层分离 |

## 3 游戏核心循环

```
进入关卡 → 击败敌人 → 升级武器 → 挑战 Boss → 获取高分 → 继续挑战
```

**设计原则：**

- **短反馈周期：** 每次击杀/拾取均有即时反馈
- **持续成长感：** 武器升级带来明显的火力提升
- **清晰视觉信息：** 弹幕颜色与玩家子弹颜色严格区分

### 3.1 每帧逻辑顺序（Game Loop）

```
1  更新输入
2  更新玩家（位置 / 状态）
3  更新敌人（AI / 移动 / 攻击）
4  更新子弹（移动 / 生命周期）
5  碰撞检测
6  清理超界对象
7  渲染帧
```

## 8 弹幕系统

### 8.1 弹幕三层架构

```
弹幕类型 → 弹幕模板 → 参数变化
```

同一算法通过调整参数即可复用，生成多种攻击形态。

### 8.2 弹幕类型总览

| 类型 | 说明 | 适用场景 | 难度 |
| --- | --- | --- | --- |
| 直线弹（Straight Shot） | 基础直线攻击 | 小型敌人 | 低 |
| 扇形弹（Fan Spread） | 覆盖角度区域 | 战斗机、坦克 | 低 |
| 环形弹（Radial Burst） | 360° 区域压迫 | 防御塔、Boss | 中 |
| 旋转弹（Rotating Ring） | 持续旋转环形 | Boss Phase 1 | 中 |
| 追踪弹（Homing Missile） | 缓慢追踪玩家 | 精英敌人、Boss | 中 |
| 波浪弹（Wave Bullet） | 正弦轨迹 | 精英单位 | 高 |
| 花形弹幕（Flower Pattern） | 多圈旋转偏移 | Boss 经典弹幕 | 高 |
| 狙击弹（Sniper Shot） | 瞬间锁定玩家 | Boss Phase 2 | 高 |

### 8.3 各弹幕算法说明

#### 扇形弹（Fan Spread）

```
参数：angleStart = -30°, angleEnd = 30°, bulletCount = 5
算法：angleStep = (angleEnd - angleStart) / (count - 1)
      for i in range(count):
          spawnBullet(angle = angleStart + i * angleStep)
```

#### 环形弹（Radial Burst）

```
参数：bulletCount = 12
算法：angleStep = 360 / count
      for i in range(count):
          spawnBullet(angle = i * angleStep)
```

#### 旋转弹幕（Rotating Ring）

```
在环形弹基础上，每帧叠加角度偏移：
angleOffset += rotationSpeed * deltaTime   // rotationSpeed = 10°/s ~ 45°/s
```

#### 狙击弹（Sniper Shot）

```
dir      = normalize(playerPos - enemyPos)
velocity = dir * speed   // speed = 12，高威胁低频率（冷却 3 s）
```

#### 波浪弹（Wave Bullet）

```
x += sin(t * freq) * amplitude * deltaTime
参数：freq = 4, amplitude = 2
```

#### 花形弹幕（Flower Pattern）

```
ringCount = 5, bulletsPerRing = 16
每圈角度偏移 += 10°（形成花朵形态）
```

#### 追踪导弹（Homing Missile）

```
targetDir  = normalize(playerPos - missilePos)
velocity  += (targetDir * speed - velocity) * steer   // steer = 0.1
```

### 8.4 敌方子弹类型参数

| 类型 | 速度 | 典型使用场景 |
| --- | --- | --- |
| 普通子弹 | 8 | 侦察机、战斗机 |
| 高速子弹 | 12 | 狙击弹、防空炮 |
| 导弹（追踪） | 4 ～ 6 | Boss、精英 |

### 8.5 弹幕性能限制

- 浏览器端最大同屏子弹数建议：**600 发**（超过会导致 FPS 下降）
- 子弹使用 `InstancedMesh` 渲染，对象池预分配 2000 个槽位
- 超出对象池上限时，优先回收距玩家最远的子弹

### 8.6 弹幕生成代码示例

```ts
// Pattern System 统一接口
spawnFanPattern({ count: 5, angleStart: -30, angleEnd: 30, speed: 6 });
spawnRadialBurst({ count: 12, speed: 4 });
spawnHomingMissile({ target: player, speed: 4, steer: 0.1 });
```

## 9 Boss 系统

每位 Boss 必须满足：**多阶段切换、攻击模式变化、至少一套标志性弹幕**。Boss HP 降至 50% 时切换阶段。

### 9.1 第一关 Boss — 巨型双炮坦克

| 属性 | 值 |
| --- | --- |
| HP | 1200 |
| 占屏比例 | 约屏幕宽度 40% |
| 进场方式 | 从屏幕顶端缓慢驶入 |

**Phase 1（HP 100% ～ 50%）**

- 双炮塔扇形弹幕：60° 扇角，每秒 20 发
- 节奏：射击 2 s → 暂停 1 s → 循环
- 旋转环形弹幕：16 发，旋转速度 20°/s

**Phase 2（HP 50% ～ 0%）**

- 装甲破损，核心暴露，进入狂暴状态
- 高速狙击弹：速度 12，冷却 1.5 s
- 波浪弹同时释放，频率 freq=4

### 9.2 第二关 Boss — 巨型变形轰炸机

| 属性 | 值 |
| --- | --- |
| HP | 1800 |
| 占屏比例 | 约屏幕宽度 70% |
| 进场方式 | 从屏幕两侧同时展开 |

**Phase 1（HP 100% ～ 65%）**

- 释放蜂群无人机：共 30 架，每架发射环形弹（8 发）

**Phase 2（HP 65% ～ 30%）**

- 花形弹幕：6 圈 × 18 发，每圈偏移 10°，覆盖全屏
- 旋转角速度 45°/s，持续高压

**Phase 3（HP 30% ～ 0%）**

- 尾部舱门打开，释放高速追踪导弹（steer=0.15，速度 6）
- 同时保持花形弹幕，形成极限压迫

### 9.3 第三关 Boss — 巨型航母

玩家需**逐个摧毁**航母的子结构，击毁顺序：雷达天线 → 防空炮群 → 导弹井 → 电磁炮核心。

| 子目标 | HP | 摧毁后效果 |
| --- | --- | --- |
| 雷达天线 | 200 | Boss 停止锁定玩家 |
| 防空炮群（×4） | 150 × 4 | 移除持续输出压制 |
| 导弹井（×2） | 300 × 2 | 停止追踪导弹 |
| 电磁炮核心 | 800 | 最终击杀 Boss |

## 10 关卡系统设计

### 10.1 关卡总结构

每关时长约 3 ～ 5 分钟，结构如下：

```
开始 → 小型敌人（热身）→ 中型敌人 → 场景事件 → 精英敌人 → Boss 前奏 → Boss 战
```

**STG 节奏设计原则：**

- 避免持续高压，循环采用：紧张 → 缓和 → 紧张 → 爆发
- 每隔约 50 s 设置一个缓和期（敌人密度 -50%，增加 PowerUp 掉落）
- Boss 出场前 20 s 清场，播放 Boss 音乐预热

### 10.2 关卡节奏曲线（以第一关为例）

```
难度
│                           Boss
│                           /\
│                          /  \
│             /\          /    \
│            /  \________/      \
│ __________/                    \____
│
└───────────────────────────────────→ 时间
  0   20  50  70       120 140 160 180s
```

| 时间段 | 敌人类型 | 弹幕 | 强度 | 备注 |
| --- | --- | --- | --- | --- |
| 0 ～ 20 s | 侦察机 | 直线弹 | 低 | 热身，玩家熟悉操作 |
| 20 ～ 50 s | 截击机 | 扇形弹 | 中 | 加入追踪敌人 |
| 50 ～ 70 s | 侦察机（减少 50%） | 直线弹 | 低 | 缓和期，大量 PowerUp 掉落 |
| 70 ～ 120 s | 坦克 + 防空炮 | 环形弹 | 中高 | 首次出现地面单位 |

### 10.7 动态难度调节

根据玩家实时表现动态调节难度，避免新手被虐、高手无挑战的两极情况：

```
difficulty = base + (playerScore / 100000) * 0.5 + (currentTime * 0.02)
```

- `difficulty` 影响：敌人生成数量、弹幕速度（±20%）、Boss 攻击间隔
- 上限：`difficulty ≤ 3.0`（防止弹幕溢出屏幕）

### 11.1 核心模块结构

```
src/
  core/
    renderer.js     // Three.js WebGPU/WebGL 渲染器初始化
```

### 11.2 渲染系统

### 11.3 碰撞系统

| 碰撞对 | 算法 | 复杂度 |
| --- | --- | --- |
| 玩家 vs 敌方子弹 | 球体碰撞（判定点半径 0.1） | O(n) |
| 玩家 vs 敌机 | 球体碰撞（模型半径 1.2） | O(n) |
| 玩家子弹 vs 敌机 | Quadtree 空间分区 | O(n log n) |
| 炸弹 vs 全屏 | 全体检测（触发时瞬间） | O(n) |

```js
// 球体碰撞判定
if (dist(a.pos, b.pos) < a.radius + b.radius) {
  handleCollision(a, b);
}
```

## 14 性能优化策略

| 优化手段 | 具体做法 | 预期收益 |
| --- | --- | --- |
| 对象池 | 子弹预分配 2000 个，粒子预分配 5000 个 | 避免 GC 卡顿 |
| GPU Instancing | 子弹、小型敌机使用 `InstancedMesh` | 渲染批次减少 90%+ |
| AI 更新节流 | AI 逻辑以 10 FPS 更新，非每帧 | CPU 节省约 30% |
| 子弹上限 | 最大同屏 600 发，超出回收最远的 | 稳定 FPS |
| 纹理图集 | 将相似纹理合并为 TextureAtlas | 减少 Draw Call |
| LOD | 远距离敌人使用低面数模型 | GPU 负载降低 |
| 视锥裁剪 | 屏幕外敌人暂停 AI 更新 | CPU 节省 |
