# ⚡ Bolt 雷电

> 基于 Three.js 驱动的 2.5D 现代化网页端弹幕射击游戏 (Shoot 'Em Up)。在浏览器中重温硬核街机时代的纯粹热血！

**Bolt** 旨在向经典街机游戏《雷电 II》致敬。它并非简单的 2D 像素复刻，而是利用现代 WebGL/WebGPU 技术进行的一场视觉与性能的全面重构。通过真实的 3D 景深、视差滚动、动态光影与基于海量粒子的弹幕演算，Bolt 在浏览器端实现了稳定 **60 FPS** 的史诗级空战体验。

## 🌍 世界观与背景

**2090 年，地球防卫战。**
名为**克拉纳斯（Cranassians）**的异星半机械生命体突袭地球。它们具备恐怖的自适应能力，瞬间寄生并接管了人类的重型坦克、航母编队与防空网络。面对铺天盖地的机械狂潮，人类启动了最后的反击兵器——**雷电战机 (Fighting Thunder ME-26)**。
作为最后的王牌飞行员，你将单机突入敌阵，从宁静的乡村农田打到高楼林立的城市废墟，最终在波涛汹涌的远洋军港迎击巨型航母核心。


## ✨ 核心游戏特性

- 🛩️ **次世代 2.5D 视觉体验**：采用 50° FOV 和 15° 倾角的透视相机，结合多层视差滚动（云层、地面、海浪），营造极具深度的战场空间感。
- 💥 **硬核弹幕 (Danmaku) 美学**：内置强大的 Pattern System，支持生成波浪、花形、旋转环形、追踪等数十种复杂弹幕，同屏 600+ 子弹无延迟。
- 🎬 **电影级后处理特效 (Post-Processing)**：基于 `UnrealBloomPass` 实现的霓虹级激光辉光，配合真实物理爆炸碎片与屏幕震动 (Screen Shake)。
- 🎼 **3D 动态音频系统**：基于 Web Audio API 构建的空间音效，引擎轰鸣、子弹擦弹音效与重金属 Boss 战 BGM 无缝交织。
- 📱 **PWA 与跨端支持**：无需下载，即开即玩。完美适配键盘、手柄，并内置移动端虚拟摇杆与自适应触控方案。


## 🎮 玩法与机制

### ⌨️ 操作指南

支持多种输入方式，自动无缝切换：

| 动作 | 键盘 | 鼠标 (推荐) | 移动端触控 |
| :--- | :--- | :--- | :--- |
| **全向移动** | `W` `A` `S` `D` 或 `↑` `↓` `←` `→` | 拖动鼠标 | 屏幕任意位置滑动 |
| **精确微操** | `Shift` + 方向键 (减速 50%) | - | - |
| **主武器射击** | `J` 键 | 自动射击 / 左键按住 | 自动射击 |
| **释放核弹** | `K` 键 | 点击右键 | 屏幕双击 |

### 🔫 三大经典武器系统 (The "Big Three")

击毁特定运输机可获得 `P` (Power-up) 道具，武器最高可升级至 8 级 (Lv.8)：

1. 🔴 **火神炮 (Vulcan - 红色)**：广角扇形散弹。Lv.8 时可覆盖前方 180° 范围，适合清理满屏的低级侦察机。
2. 🔵 **高能激光 (Laser - 蓝色)**：直线贯穿光束。DPS 极高，对大型机甲和 Boss 具有毁灭性打击，命中时产生强烈的蓝色辉光飞溅。
3. 🟣 **诱导激光 (Plasma - 紫色)**：标志性的“雷电式”软鞭激光。基于贝塞尔曲线算法实现，自动锁定并扭曲追踪全屏敌人，Lv.8 时可同时锁定 5 个目标。

### 🏅 核心高分机制
- **擦弹 (Graze)**：战机的真实受击判定点（Hitbox）仅为中心的**极小发光球体 (半径 0.1)**。让敌方子弹擦过机身（不触碰核心），可获得极大的操作快感。
- **勋章连击**：击毁敌方建筑会掉落勋章。静止拾取 500 分，飞行移动中拾取 1000 分。连续拾取不中断，勋章会逐渐变为金色、彩色，单个得分飙升至 100,000 分。


## 🛠️ 技术栈与架构

项目采用纯现代前端技术栈构建，零传统 DOM 游戏负担：

- **核心引擎**: [Three.js](https://threejs.org/) (WebGL / WebGPU 渲染)
- **构建与热更新**: [Vite](https://vitejs.dev/)
- **开发语言**: [TypeScript](https://www.typescriptlang.org/) (严格类型，保障复杂物理运算的稳定性)
- **状态与逻辑**: 自定义轻量级 ECS (Entity-Component-System) 架构思想
- **模型与动画**: GLTF/GLB 格式，使用 Blender 预渲染烘焙

### 📂 核心目录结构

```text
bolt-reborn/
├── public/                 # 静态挂载资源 (网站 Icon, manifest.json)
├── src/
│   ├── assets/             # 游戏内资源
│   │   ├── models/         # 敌机、自机、Boss 的 .glb 模型
│   │   ├── textures/       # 滚动的地面、云层、粒子贴图
│   │   └── audio/          # BGM 与 3D 空间音效
│   ├── core/               # 引擎核心层
│   │   ├── Engine.ts       # Main Loop (requestAnimationFrame)
│   │   ├── Renderer.ts     # WebGLRenderer 配置与 Post-Processing 管线
│   │   └── Camera.ts       # 2.5D 透视相机逻辑与震动算法
│   ├── entities/           # 游戏实体
│   │   ├── Player.ts       # 玩家控制与状态机
│   │   ├── Enemy.ts        # 敌机基类 (侦察机, 战斗机, 重装坦克)
│   │   └── Boss.ts         # Boss 多阶段状态机
│   ├── systems/            # 核心业务系统
│   │   ├── BulletManager.ts# Pattern System (弹幕工厂)
│   │   ├── Collision.ts    # 基于 Quadtree 的球体/射线碰撞检测
│   │   └── ObjectPool.ts   # 泛型对象池 (极度重要，防止 GC)
│   ├── config/             # 数据驱动层 (JSON配置)
│   │   └── Level_1.json    # 基于时间轴的关卡生成脚本
│   └── main.ts             # 游戏入口点
├── index.html              # 宿主 HTML
└── package.json            
````

## 🚀 快速开始 (本地开发)

### 1\. 环境准备

  - [Node.js](https://nodejs.org/) (推荐 v18.x 或更高版本)
  - 支持 WebGL 2.0 的现代浏览器

### 2\. 克隆与安装

```bash
# 克隆仓库
git clone [https://github.com/your-org/bolt-reborn.git](https://github.com/your-org/bolt-reborn.git)
cd bolt-reborn

# 安装依赖 (推荐使用 pnpm 或 npm)
npm install
```

### 3\. 本地运行

```bash
# 启动包含 HMR (热更新) 的本地开发服务器
npm run dev
```

打开浏览器访问控制台输出的地址（通常是 `http://localhost:5173`）。

### 4\. 生产构建

```bash
# 编译 TypeScript 并打包压缩资源
npm run build

# 在本地预览打包后的生产版本
npm run preview
```


## 📈 性能优化方案

为了在复杂的弹幕计算下保持 **60 FPS**（即使在中低端设备或移动端上），我们在引擎底层做了以下极限优化：

1.  **零 GC 策略 (Object Pooling)**
    游戏主循环 (`GameLoop`) 内**绝对禁止**使用 `new` 关键字分配内存。所有子弹（2000 个槽位）、爆炸粒子（5000 个槽位）和敌机均在游戏初始化时预先分配至 `ObjectPool`，死亡后回收到池中，彻底消灭因垃圾回收 (Garbage Collection) 导致的突发性卡顿。
2.  **GPU Instancing 渲染**
    敌方散弹和小型无人机使用 `THREE.InstancedMesh` 进行批量渲染。即使屏幕上有 1000 颗子弹，Draw Call 依然保持在个位数。
3.  **AI 逻辑降频 (Throttling)**
    敌机的寻路、索敌 AI 不需要每帧 (60次/秒) 运算。系统将 AI 更新频率锁定在 **10 FPS**，并通过线性插值 (Lerp) 保证视觉平滑，为 CPU 节约了超过 30% 的算力。
4.  **空间分区碰撞 (Quadtree)**
    摒弃了暴力的 $O(N^2)$ 遍历，将屏幕划分为动态四叉树。仅计算同一象限内的子弹与机体碰撞，配合简单的 `Radius` 球面交叉算法，使碰撞计算开销微乎其微。


## 🗺️ 开发路线图

我们采用里程碑式的迭代方式，目前正处于 **Alpha 测试阶段**：

  - [x] **M1: 引擎点火 (Engine Ignition)** - 跑通 Three.js 渲染管线，实现地表无限视差滚动与玩家平滑输入操作。
  - [x] **M2: 火力全开 (Firepower)** - 完成红、蓝、紫三大武器的逻辑开发，实现对象池与基础圆形碰撞检测。
  - [ ] **M3: 弹幕地狱 (Bullet Hell) [当前进行中]** - 解析 `Level.json` 关卡脚本，实装 24 种敌机 AI 和弹幕生成算法 (Pattern System)。
  - [ ] **M4: 终极决战 (Boss Rush)** - 第一至第三关巨型 Boss 多形态开发，集成 Web Audio API 动态音效。
  - [ ] **M5: 移动端与发布 (Deployment)** - 触控摇杆适配、UI/HUD 美化、接轨全球排行榜并部署至 GitHub Pages/Vercel。
